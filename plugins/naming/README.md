# Naming

앱·서비스 이름을 짓는 플러그인입니다. 네이밍 이론(유형·조어 기법·평가 프레임워크)에 기반해 4개 전략별 에이전트가 후보를 생성하고, 평가 에이전트가 랭킹하며, 가용성 에이전트가 도메인·상표·다국어 의미를 점검합니다.

## 설치

```
/plugin install naming@ben3329-plugins
```

## 사용

```
/naming:naming 친구끼리 더치페이를 자동 정산해주는 앱
```

컨셉 한 줄만 줘도 나머지 브리프(타깃·톤·언어권 등)를 추정해 채운 뒤 확정 브리프를 보여주고 진행합니다. 인자가 비어 있으면 컨셉·타깃·톤·언어권을 먼저 물어봅니다. `.md` 브리프 파일 경로도 받습니다.

## 동작

1. **브리프 확정** — 컨셉·편익·타깃·톤·언어권·경쟁군·카테고리·확장범위·주표기·선호·금지 11필드
2. **4개 생성 에이전트 병렬 실행** (전략별 후보 8~12개씩)
   - **nm-descriptive** — 서술형·암시형 (의미·편익 기반: PayPal, Slack류)
   - **nm-coined** — 조어형 (혼성어·오탈자·접사·신조어: Pinterest, Spotify, Verizon류)
   - **nm-metaphor** — 은유·상징·추상·어근 (Amazon, Nike, Lumen류)
   - **nm-experimental** — 실험적·음성상징·짧은 임팩트 (Zoom, Bolt류)
3. **후보 풀 취합·중복 제거**
4. **평가 에이전트** (nm-evaluator) — Neumeier 7기준(0/3/5점 앵커) + SMILE/SCRATCH로 채점·랭킹, 숏리스트 선정
5. **가용성 에이전트** (nm-availability) — 숏리스트 대상으로 도메인(WHOIS)·상표(KIPRIS/USPTO/WIPO, 카테고리 기반 Nice 류 추정)·앱스토어/핸들·다국어 의미 점검(🟢/🟡/🔴), 공식 출처 우선
6. **추천 리포트 출력** — 점수 Top + 가용성 반영 최종 추천(🔴는 강등) + 전체 랭킹 + 전략별 후보 부록 + 다음 단계 체크리스트

## 평가 기준

- **Neumeier 7기준** (《The Brand Gap》): 차별성 · 간결성 · 적합성 · 발음/철자 용이성 · 호감도 · 확장성 · 보호 가능성
- **SMILE / SCRATCH** (《Hello, My Name is Awesome》): 좋은 이름의 5요소(SMILE) 가점 / 피해야 할 7요소(SCRATCH) 감점

## 주의

가용성 점검은 웹 검색 기반 **참고 단서**이며 법적 보증이 아닙니다. 이름 최종 확정 전에 도메인(WHOIS), 상표(KIPRIS/USPTO/WIPO) 정식 조회와 현지화 검수가 필요합니다.
