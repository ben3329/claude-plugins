기획서 파일을 4개 전문 에이전트가 병렬로 리뷰하고 결과 리포트를 출력합니다. **자동 수정은 하지 않습니다** — 기획서는 사용자의 의사결정이 강하게 반영된 문서이므로 발견 사항만 보고하고 사용자가 직접 반영합니다.

인자: `$ARGUMENTS` — 기획서 `.md` 파일 경로 또는 폴더 경로

## 워크플로

### 1단계: 대상 파일 결정

`$ARGUMENTS`로부터 리뷰할 `.md` 파일을 결정합니다:

1. 인자가 비어있으면 사용자에게 경로를 요청하고 중단합니다.
2. 인자가 `.md` 파일이면 그대로 사용합니다.
3. 인자가 폴더면 다음 순서로 탐색합니다:
   - `planning-draft.md`, `spec.md`, `<폴더명>.md`
   - 그래도 없으면 폴더 내 `.md` 파일이 정확히 하나일 때 그것을 사용합니다.
   - 후보가 여럿이면 목록을 보여주고 사용자에게 선택을 요청한 뒤 중단합니다.
4. 파일이 존재하는지 확인합니다. 없으면 사용자에게 알리고 중단합니다.

표시: `## 기획서 리뷰: <경로>`

### 2단계: 참조 문서 수집

기획서 본문에서 마크다운 링크 `[...](상대경로.md)` 형식의 **경로 부분만** 추출합니다. Bash 예시:

```bash
grep -oE '\]\([^)]+\.md[^)]*\)' <기획서경로> | sed -E 's/^\]\((.*)\)$/\1/' | sort -u
```

추출된 경로는 기획서 파일 기준 상대 경로이므로 절대 경로로 변환하여 에이전트 프롬프트에 전달합니다 (기획서 디렉터리 + 상대 경로). 참조가 없으면 "없음" 문자열을 전달합니다.

### 3단계: 4개 에이전트 병렬 실행

ALL 4개 에이전트를 단일 메시지에 다중 Agent 호출로 동시에 실행합니다. 각 에이전트의 `subagent_type`은 아래 목록의 `plan-review:pr-*` 형식(플러그인명:에이전트명)을 그대로 사용합니다.

실행할 에이전트:
1. `plan-review:pr-clarity` — 명확성
2. `plan-review:pr-completeness` — 완결성
3. `plan-review:pr-consistency` — 일관성
4. `plan-review:pr-feasibility` — 실행가능성

아래 프롬프트 템플릿에서 `<경로>`는 1단계에서 결정한 기획서 파일의 절대 경로로, `<참조 경로 목록>`은 2단계에서 수집한 절대 경로 목록(또는 "없음")으로 **치환한 뒤** 각 에이전트에 전달합니다 (플레이스홀더를 리터럴로 보내지 마세요).

```
다음 한글 기획서를 리뷰해주세요. 4개 에이전트가 병렬로 리뷰 중이므로 본인 도메인 외 이슈는 절대 보고하지 마세요:
- pr-clarity: 명확성 (모호한 표현, 미정의 용어, 회피 표현)
- pr-completeness: 완결성 (누락 엣지케이스, 에러 처리, 화면 상태)
- pr-consistency: 일관성 (내부 모순, 용어 불일치, 참조 문서 정합성)
- pr-feasibility: 실행가능성 (데이터 모델, 권한 모델, 기술 제약)

기획서 경로: <경로>
참조 문서 경로 목록: <참조 경로 목록>

기획서 전문은 Read 도구로 직접 읽어주세요. 참조 문서는 일관성 검토에 필요한 경우에만 읽으세요.
```

### 4단계: 결과 집계 및 리포트 출력

4개 에이전트의 finding을 모두 수집하여 다음 형식으로 한국어로 출력합니다.

- 신뢰도 50 미만 finding은 출력에서 제외합니다 (소음 감소).
- 신뢰도 80 이상은 ★ 표시로 강조합니다.
- 에이전트 Finding 필드를 다음과 같이 매핑합니다:
  - **한 줄 요약**: 에이전트 출력의 `문제` / `누락 사항` / `모순` 필드의 첫 문장 (없으면 카테고리명)
  - **위치**: `위치` 필드 (또는 `위치 A` / `위치 B`)를 그대로 옮김
  - **문제 / 제안**: 에이전트 출력의 해당 필드 그대로
  - 에이전트 출력에 없는 필드는 생략

```
## 리뷰 결과

총 X건 (명확성 N건 / 완결성 N건 / 일관성 N건 / 실행가능성 N건)

### 명확성 (pr-clarity)

#### ★ [신뢰도: 90] §3.2 — 한 줄 요약
- **위치**: 섹션/줄 번호
- **문제**: ...
- **제안**: ...

(다음 finding...)

### 완결성 (pr-completeness)
...

### 일관성 (pr-consistency)
...

### 실행가능성 (pr-feasibility)
...
```

각 차원에 finding이 없으면 "이슈 없음"으로 표시합니다.

마지막에 우선순위 요약을 한 줄 추가합니다:

```
**우선 검토 권장**: ★ 표시된 N건부터 확인하세요.
```

## 규칙

- 모든 사용자 출력은 한국어로 합니다.
- 에이전트 finding은 한국어로 반환되며, 오케스트레이터는 위 매핑 규칙에 따라 재정리하여 출력합니다.
- 참조 문서 목록이 "없음"이거나 일부 파일을 읽지 못해도 진행합니다. 일관성 finding의 참조 문서 카테고리만 스킵됩니다.
