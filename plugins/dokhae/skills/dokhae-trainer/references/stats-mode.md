# 기록·통계 모드 (stats)

누적 학습 기록(`records.jsonl`)을 **분석**하는 모드입니다. **지문을 생성하지 않으며**,
기존 기록만 읽어 추이·약점·추천을 제시합니다. 공통 규칙은 [`_spec.md`](_spec.md)를 따릅니다.

> 어조: 한국어 존댓말. 총평은 빈 칭찬 없이 **수치 근거**로 씁니다. 출력은 표 위주로 간결하게.

---

## 0. 입력 파일

- 경로: **`$HOME/.claude/dokhae/records.jsonl`**
- 형식: JSON Lines(한 줄 = 한 세션). 스키마는 `_spec.md` 7절과 동일합니다.
- 모드별로 해당 없는 필드는 `null`이므로 **모드로 먼저 거른 뒤** 집계합니다.

---

## 1. 기록 읽기

1. 파일 존재 여부를 확인합니다.
   ```bash
   test -f "$HOME/.claude/dokhae/records.jsonl" && wc -l "$HOME/.claude/dokhae/records.jsonl"
   ```
2. **파일이 없거나 0줄(빈 파일)** 이면 분석하지 않고 아래만 안내합니다.
   > "아직 기록이 없습니다. `/dokhae:dokhae quiz 3` 같은 훈련을 한 세션 진행하면 다음부터 통계를 보여드릴 수 있습니다."
   그리고 종료합니다.
3. 기록이 있으면 **모든 줄**을 읽습니다.
   ```bash
   cat "$HOME/.claude/dokhae/records.jsonl"
   ```
4. **Claude가 각 줄을 직접 JSON으로 파싱**해 메모리에서 집계합니다(아래 2). `jq`는 **있으면 보조**로만
   쓰고(예: `jq -s '.' …`), 없거나 실패해도 직접 파싱으로 진행합니다.
5. **깨진 줄**(파싱 실패)은 건너뛰고, 마지막 총평에서 "n줄 무시"로만 짧게 언급합니다.

---

## 2. 집계 항목

아래를 계산합니다. 분모가 0인 항목은 `—`로 표기합니다.

| 항목 | 계산 |
|------|------|
| 총 세션 수 | 유효한 줄 수 |
| 모드별 횟수 | `mode`별 카운트(`quiz`/`summary`/`speed`) |
| quiz 평균 정답률 | `quiz` 줄의 `accuracy` 평균 |
| speed 평균 cpm | `speed` 줄의 `cpm` 평균 |
| speed 평균 이해도 | `speed` 줄의 `comp_accuracy` 평균 |
| summary 3축 평균 | `summary` 줄 `scores`의 `accuracy`/`completeness`/`concision` 각 평균(0–100) |
| 영역별 quiz 정답률 | `area`별 **quiz** `accuracy` 평균(주 지표). speed `comp_accuracy`·summary 점수는 측정 대상이 달라 **합산하지 않고** 별도 표기 |
| 최빈 약점 태그 Top | 모든 줄 `weak_tags`를 펼쳐 빈도 집계, 상위 3개 |

- 영역(`area`)은 `_spec.md` 3절 5개(`인문`/`사회`/`과학`/`기술`/`예술`)로만 묶습니다.
- 약점 태그는 `_spec.md` 5절 표준 10개(`사실적이해`/`추론`/`구조파악`/`어휘문맥`/`비판적용`/
  `핵심누락`/`지엽집착`/`왜곡과장`/`속도부족`/`정확도부족`)로만 집계합니다.

---

## 3. 추이 계산 (개선/하락)

"추이"는 **최근 N개 vs 그 이전**의 단순 비교로 표시합니다.

1. 모드별로 시간순 정렬(`date`+`time` 기준) 후, 해당 모드 줄을 **최근 N개**와 **그 이전 전체**로 나눕니다.
   - 기준 지표: quiz=`accuracy`, speed=`cpm`(보조로 `comp_accuracy`), summary=3축 평균.
   - N 기본값 = **3**. 모드 줄이 4개 미만이면 "표본 부족"으로 추이 생략.
2. 두 구간의 평균 차이를 화살표로 표기합니다.
   - 개선 ▲ / 하락 ▼ / 유지 ▬ (임계: quiz·이해도는 ±0.05, cpm은 ±5%, summary는 ±3점).

---

## 4. 출력 형식

표 위주로, 아래 4개 블록을 순서대로 출력합니다. (값이 없는 행은 생략 가능)

### A. 개요
| 항목 | 값 |
|------|----|
| 총 세션 | (수) |
| 모드별 | quiz n · summary n · speed n |
| 분석 기간 | (첫 날짜) ~ (마지막 날짜) |

### B. 모드별 성과 + 추이
| 모드 | 평균 | 최근 N 추이 |
|------|------|-------------|
| quiz | 정답률 0.00 | ▲/▼/▬ |
| speed | cpm 000 · 이해도 0.00 | ▲/▼/▬ |
| summary | 정확 00 / 완전 00 / 간결 00 | ▲/▼/▬ |

### C. 영역별 quiz 정답률 (약한 순)
| 영역 | 세션 | quiz 정답률 |
|------|------|-------------|
| (가장 약한 영역부터) | | |

> speed 이해도·summary 점수는 quiz 정답률과 섞지 않습니다. 영역별 표본이 충분하면 별도 행/열로,
> 적으면 생략하고 총평에서만 언급합니다.

### D. 최빈 약점 Top
| 순위 | 약점 태그 | 횟수 |
|------|-----------|------|
| 1~3 | | |

이어서 **총평 한두 줄**: 가장 두드러진 강·약점을 수치로 짚습니다(빈 칭찬 금지).

---

## 5. 추천 (한 줄)

가장 약한 영역과 최빈 약점 태그를 결합해 **다음 추천 한 줄**을 제시합니다.

- 형식: `다음 추천 → /dokhae:dokhae <모드> <난이도> (영역: <area>) — <약점태그> 보강`
- 모드 선택 가이드:
  - 약점 태그가 `사실적이해`/`추론`/`구조파악`/`어휘문맥`/`비판적용` → **quiz**
  - `핵심누락`/`지엽집착`/`왜곡과장` → **summary**
  - `속도부족`/`정확도부족` → **speed**
- 난이도: 해당 영역 평균 정답률이 0.8↑이면 +1, 0.5↓이면 −1, 그 외 직전 난이도 유지(범위 1–5).

---

## 6. 예시 출력 (가상 데이터)

> 입력: `records.jsonl` 9줄(quiz 5 · summary 2 · speed 2), N=3.

**A. 개요**

| 항목 | 값 |
|------|----|
| 총 세션 | 9 |
| 모드별 | quiz 5 · summary 2 · speed 2 |
| 분석 기간 | 2026-06-12 ~ 2026-06-30 |

**B. 모드별 성과 + 추이**

| 모드 | 평균 | 최근 3 추이 |
|------|------|-------------|
| quiz | 정답률 0.72 | ▲ (0.66 → 0.78) |
| speed | cpm 612 · 이해도 0.58 | ▬ |
| summary | 정확 78 / 완전 64 / 간결 81 | ▼ (완전성 하락) |

**C. 영역별 quiz 정답률 (약한 순)**

| 영역 | 세션 | quiz 정답률 |
|------|------|-------------|
| 사회 | 2 | 0.55 |
| 과학 | 3 | 0.70 |
| 인문 | 2 | 0.83 |

**D. 최빈 약점 Top**

| 순위 | 약점 태그 | 횟수 |
|------|-----------|------|
| 1 | 추론 | 4 |
| 2 | 핵심누락 | 3 |
| 3 | 정확도부족 | 2 |

**총평**: quiz 정답률은 최근 0.66→0.78로 올랐으나, `사회` 영역(0.55)과 `추론` 문항(4회 오답)이
일관된 약점입니다. 속독은 cpm 612로 빠르지만 이해도 0.58로 `정확도부족`이 동반됩니다.

**추천**: `다음 추천 → /dokhae:dokhae quiz 3 (영역: 사회) — 추론 보강`
(사회 평균 0.55로 난이도 유지, 추론 약점이 최빈이므로 quiz 선택)

---

## 7. 주의

- 이 모드는 **기록을 변경하지 않습니다**(읽기 전용). 새 줄을 append하지 않습니다.
- 표준 외 영역·태그가 보이면 집계에서 제외하고 총평에서 "비표준 항목 무시"로만 언급합니다.
- 모든 평균은 소수 둘째 자리(정답률·이해도) 또는 정수(cpm·점수)로 반올림해 표기합니다.
