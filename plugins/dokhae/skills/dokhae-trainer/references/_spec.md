# 독해 훈련 공유 사양 (_spec.md)

모든 모드(문제풀이·요약·속독)와 기록·통계가 공유하는 **단일 기준 문서**다.
다른 reference 파일들은 이 문서를 그대로 따른다. 충돌이 있으면 이 문서가 우선한다.

---

## 1. 어조·언어 규칙

- 출력 언어는 **한국어**, 기본 **존댓말**.
- 학습자에게는 동기를 주되, **채점은 엄정하고 구체적**으로 한다. "잘했어요" 같은 빈 칭찬 대신
  **무엇이 왜 맞고 틀렸는지 근거**를 제시한다.
- 군더더기·반복 설명을 피하고, 표·번호 목록으로 가독성을 확보한다.
- 지문·문제는 학습자가 한 화면에서 읽기 좋게 **명확히 구분**해 제시한다(아래 4 참조).

---

## 2. 난이도 척도 (level 1–5)

| level | 이름 | 분량(공백 제외 한글 기준) | 특징 |
|-------|------|---------------------------|------|
| 1 | 입문 | 350–500자 / 1–2문단 | 친숙한 주제, 쉬운 어휘, 직접 진술 위주, 추론 거의 없음 |
| 2 | 기초 | 500–700자 / 2문단 | 약간의 추론, 한두 개의 개념어 |
| 3 | 표준(수능 평이) | 800–1000자 / 3–4문단 | 추상 개념 1개, 전문용어 약간, 명시적 논리 구조 — **기본값** |
| 4 | 심화(수능 고난도) | 1100–1400자 / 4–5문단 | 복합 논증, 대조·인과 구조, 전문용어 다수, 함정 선지 |
| 5 | 최상 | 1500–1900자 / 5문단+ | 다층 논증, 여러 관점 비교, 높은 추상도, 정교한 추론 요구 |

- 모드 시작 시 level이 지정되지 않으면 **3(표준)** 으로 시작한다.
- 학습자가 "더 쉽게/어렵게"를 요청하면 ±1 조정한다.

---

## 3. 주제 영역(area) 분류

지문은 **비문학(설명문·논설문)** 이며 아래 5개 영역에서 고른다.

| area 코드 | 영역 | 세부 예시 |
|-----------|------|-----------|
| `인문` | 인문 | 철학·윤리·역사·논리·언어 |
| `사회` | 사회 | 경제·법·정치·미디어·사회학 |
| `과학` | 과학 | 물리·화학·생명·지구·천문 |
| `기술` | 기술 | 공학·IT·재료·정보통신 |
| `예술` | 예술 | 미학·음악·미술·건축 |

**영역 선택 규칙**
1. 학습자가 영역을 지정하면 그대로 따른다.
2. 기록(records.jsonl)이 있으면 최근 세션 중 정답률·이해도·요약 점수가 낮거나 약점 태그가 나온 세션의 `area`를 우선 검토한다.
3. 직전 2개 세션과 **같은 영역 연속 반복을 피한다**(다양성 확보).

---

## 4. 지문 작성 규칙

- 형식: 한국어 **비문학 지문**. 명확한 **중심 화제 1개** + 논리 구조 1개
  (정의-예시-함의 / 주장-근거-반론 / 비교-대조 / 원인-결과 / 통념-반박 등).
- **자기완결성**: 모든 문제의 정답 근거가 **지문 안에서** 확인 가능해야 한다.
  채점은 외부 상식이 아니라 "지문에 근거"로 판단한다.
- **사실 정합성**: 실제로 타당한 내용으로 쓴다. 검증 가능한 사실 오류를 넣지 않는다.
- 문단은 채점 시 근거를 지칭할 수 있도록 **`[1] [2] [3]`** 처럼 문단 번호를 붙여 제시한다.
- 분량·난이도는 위 2의 척도를 지킨다. 분량은 ±15% 허용.
- 정답·해설·문제는 **지문을 제시할 때 절대 함께 노출하지 않는다**(아래 6 참조).

---

## 5. 약점 태그(weak_tags) 표준 어휘

모드를 가로질러 통계를 집계하기 위해, 약점은 **아래 표준 태그**로만 기록한다.

| 태그 | 의미 |
|------|------|
| `사실적이해` | 지문에 명시된 세부 정보를 잘못 읽음 |
| `추론` | 명시되지 않은 내용을 근거로 추론하지 못함 |
| `구조파악` | 글의 전개·문단 관계·논증 구조를 놓침 |
| `어휘문맥` | 문맥적 의미·지시어·어휘를 오해함 |
| `비판적용` | 주장 평가·사례 적용·반론 판단을 못함 |
| `핵심누락` | 요약 시 중심 내용을 빠뜨림 |
| `지엽집착` | 지엽적 정보를 핵심으로 오인함 |
| `왜곡과장` | 지문에 없는 내용을 더하거나 과장·왜곡함 |
| `속도부족` | 목표 속도(CPM) 대비 느림 |
| `정확도부족` | 속도는 났으나 이해도가 낮음 |

## 6. 노출 통제(중요)

- 지문을 보여줄 때는 **지문만** 보여준다. 채점용 **개별 문항·선지·정답·해설·모범답안**은 같은 메시지에 넣지 않는다.
- 채점 전까지 모범답안·정답을 절대 공개하지 않는다(학습자가 스스로 풀게 한다).
- 속독 모드에서는 채점용 이해도 문제를 **읽기 시간 측정이 끝난 뒤에만** 제시한다.
- **예외**: 요약 모드처럼 *과제 지시 자체가 읽기 방식*인 경우, "읽고 요지·문단 핵심을 정리해 주세요" 수준의
  **일반 과제 지시**는 지문과 같은 메시지에 둘 수 있다. 그러나 quiz의 개별 문항·선지, speed의 이해도 문제처럼
  **읽기 방향을 미리 편향시키는 구체적 문항**은 읽기 전/지문과 함께 노출하지 않는다.
- **기준 확정 ≠ 공개**: 정답키·모범답안·채점 기준은 *문항/지문 생성 시점에 내부적으로 확정*해 두고,
  **공개만** 채점 후로 미룬다. 학습자 답을 본 뒤에 기준을 만들면 사후합리화로 채점이 흔들리므로 금지한다.

## 7. 기록 스키마 (records.jsonl)

- 저장 위치: **`$HOME/.claude/dokhae/records.jsonl`** (없으면 `mkdir -p` 후 생성).
- 한 세션 = **JSON 한 줄**(JSON Lines). 줄 끝에 개행.
- 통합 스키마(모드별로 해당 없는 필드는 `null`):

```json
{
  "date": "2026-06-30",
  "time": "15:42",
  "mode": "quiz",
  "area": "과학",
  "topic": "엔트로피와 열역학 제2법칙",
  "level": 3,
  "char_count": 920,
  "q_count": 5,
  "correct": 4,
  "accuracy": 0.8,
  "scores": null,
  "elapsed_sec": null,
  "cpm": null,
  "comp_q": null,
  "comp_correct": null,
  "comp_accuracy": null,
  "weak_tags": ["추론"],
  "note": "3번 추론 문제에서 반례를 놓침"
}
```

필드 정의:

| 필드 | 모드 | 의미 |
|------|------|------|
| `date` / `time` | 공통 | 날짜(YYYY-MM-DD) / 시각(HH:MM). `date "+%Y-%m-%d %H:%M"`로 취득 |
| `mode` | 공통 | `"quiz"` \| `"summary"` \| `"speed"` |
| `area` / `topic` / `level` / `char_count` | 공통 | 영역 / 소재 / 난이도 / 지문 글자수(공백 제외) |
| `q_count` / `correct` / `accuracy` | quiz | 문항 수 / 정답 수 / 정답률(0–1) |
| `scores` | summary | `{"accuracy":0-100,"completeness":0-100,"concision":0-100}` |
| `elapsed_sec` / `cpm` | speed | 읽기 소요(초) / 분당 글자수(char_count ÷ (elapsed_sec/60)) |
| `comp_q` / `comp_correct` / `comp_accuracy` | speed | 이해도 문항 수 / 정답 수 / 이해도(0–1) |
| `weak_tags` | 공통 | 위 5의 표준 태그 배열 |
| `note` | 공통 | 한 줄 메모(무엇을 왜 틀렸는지 핵심) |

---

## 8. 기록 적재 절차 (모든 모드 공통, 채점 후)

1. `mkdir -p "$HOME/.claude/dokhae"` (이미 있으면 무해).
2. 현재 시각 취득: `date "+%Y-%m-%d %H:%M"`.
3. 세션 결과를 위 스키마 한 줄(JSON)로 만들어 `records.jsonl`에 **append**한다.
   - 안전하게: `cat >> "$HOME/.claude/dokhae/records.jsonl" <<'EOF'` 형태의 quoted here-doc으로 JSON 한 줄과 개행을 추가한다. `echo '...'`처럼 JSON 전체를 작은따옴표로 감싸지 않는다.
4. 적재 후 학습자에게 "기록됨"을 짧게 알리고, **약점 태그 1–2개**를 요약해 다음 훈련 포커스를 제안한다.
